## ADDED Requirements

### Requirement: Ensemble world model training
The system SHALL train an ensemble of 3-5 world models with shared encoder weights but separate predictor heads, enabling uncertainty estimation through prediction variance.

#### Scenario: Train ensemble predictors
- **WHEN** training data is available
- **THEN** the system trains N separate predictor heads on the same encoded latents, each with independent weight initialization

#### Scenario: Shared encoder forward pass
- **WHEN** encoding frames for ensemble prediction
- **THEN** the system runs the shared encoder once and feeds the resulting latents to all predictor heads

### Requirement: Uncertainty estimation
The system SHALL compute uncertainty estimates for world model predictions as the variance (or standard deviation) across ensemble member predictions.

#### Scenario: Compute prediction uncertainty
- **WHEN** an ensemble of world models predicts the next latent for the same input
- **THEN** the system returns the mean prediction and the variance across ensemble members

#### Scenario: Per-step uncertainty in rollout
- **WHEN** performing a latent rollout with uncertainty tracking
- **THEN** the system returns cumulative uncertainty (sum of per-step variances) alongside the predicted trajectory

### Requirement: Safe plan selection
The system SHALL filter candidate plans generated by the planner based on cumulative uncertainty, discarding plans that exceed a configurable uncertainty threshold.

#### Scenario: Discard high-uncertainty plans
- **WHEN** the planner generates candidate action sequences and evaluates them with the ensemble
- **THEN** the system discards candidates whose cumulative rollout uncertainty exceeds the threshold

#### Scenario: Prefer conservative plans
- **WHEN** multiple candidate plans reach the goal with similar cost
- **THEN** the system selects the plan with the lowest cumulative uncertainty

#### Scenario: Fallback when all plans are uncertain
- **WHEN** all candidate plans exceed the uncertainty threshold
- **THEN** the system returns the plan with the lowest uncertainty and flags it as low-confidence

### Requirement: Safety constraint checking
The system SHALL support pluggable safety constraints that evaluate candidate trajectories and reject those that violate defined safety bounds.

#### Scenario: Apply safety constraints to candidates
- **WHEN** candidate trajectories are evaluated during planning
- **THEN** the system checks each trajectory against all registered safety constraints and rejects violations

#### Scenario: Custom constraint registration
- **WHEN** a user registers a custom safety constraint function
- **THEN** the system includes that constraint in the evaluation of all future candidate plans

### Requirement: Safety configuration
The system SHALL support configurable safety parameters including ensemble size, uncertainty threshold, and constraint definitions.

#### Scenario: Custom ensemble size
- **WHEN** an ensemble size of N is configured (3 ≤ N ≤ 5)
- **THEN** the system trains exactly N predictor heads

#### Scenario: Configuration validation
- **WHEN** invalid parameters are provided (e.g., ensemble size < 2, negative threshold)
- **THEN** the system raises a validation error
