## 1. Implement RelayBuffer and RelayHTTPServer in `src/agent/relay.py`

- [x] 1.1 Create `src/agent/relay.py` with a `RelayBuffer` class that holds the latest `(jpeg_bytes, bgr_ndarray, timestamp)` triple behind a `threading.Lock`. Constructor takes `max_width` (default 640), `max_height` (default 360), `jpeg_quality` (default 80). `update(pov_rgb)` does RGB→BGR conversion, letterboxed resize to fit `max_width × max_height` preserving aspect, JPEG-encodes, and atomically swaps the slot. `snapshot()` returns a deep copy of the BGR ndarray and a reference to the cached JPEG bytes.
- [x] 1.2 Add a `RelayHTTPServer` class in the same file that wraps `http.server.ThreadingHTTPServer` + a custom `BaseHTTPRequestHandler`. The handler routes `GET /stream` to a generator that yields `multipart/x-mixed-replace` chunks at `min_frame_interval_ms` (default 33) and routes `GET /healthz` to `200 ok\n`. Other paths return 404.
- [x] 1.3 Add a `start()` method that launches the server in a daemon `threading.Thread` and a `stop()` method that calls `server.shutdown()` + `server.server_close()`. Constructor takes `(host, port, buffer, min_frame_interval_ms)`.
- [x] 1.4 Add unit tests in `tests/test_relay.py`: `RelayBuffer` round-trip (write RGB → snapshot BGR matches `cv2.cvtColor`), `update(None)` clears the slot, downsampling preserves aspect, `update` from one thread + `snapshot` from another does not deadlock, and `RelayHTTPServer` responds correctly to `/healthz` and `/stream` (verify with a `urllib.request` test client using a background thread that pushes frames).
